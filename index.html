2024年5月

核心职责、内容与成果：

  • 虚拟机环境搭建：安装 RHEL8 操作系统，配置本地镜像源（BaseOS），通过 yum.repos.d 配置文件设置 baseurl 为 file:///mnt/cdrom/BaseOS，启用 repo 源（enabled=1）并关闭 GPG 校验（gpgcheck=0）
  • 网络与服务配置：配置静态 IP 地址，确保虚拟机网络连通，安装 httpd 服务，修改主配置文件 httpd.conf，调整 Listen 端口与 ServerRoot 等核心参数

  • 服务验证与优化：启动 httpd 服务并设置开机自启，通过本地浏览器访问虚拟机 IP 地址，成功加载网站内容，完成服务稳定性测试

ansible网络自动化运维

2025年3月

搭建 Ansible 自动化运维平台，通过编写 Playbook 脚本实现多主机统一管理，替代传统手动运维模式，提升运维效率与配置一致性

核心职责、内容与成果：

  • 环境搭建与配置：完成 Ansible 主从架构部署，配置主机间免密登录，编写 inventory 主机清单，实现 master 节点对 node1、node2 等节点的统一管控
  • 自动化脚本开发：编写 NFS 服务部署 Playbook，通过 yum 模块安装 nfs-utils，利用 systemd 模块重启 firewalld 与 nfs-server 服务，配置防火墙永久开放 nfs、rpc-bind、mountd 服务端口
  • 共享存储配置：在 node1 节点创建 /mnt/test 共享目录，编写 /etc/exports 配置文件实现目录共享，通过 exportfs -r 命令应用配置，在 node2 节点成功挂载共享目录 /mnt/exam
  • 主机信息自动化采集：开发 configure_host.yml 脚本，利用 Jinja2 模板遍历所有主机，采集并输出各节点的 IP 地址与主机名，生成标准化配置文件 /tmp/hosts.txt
  • 项目成效：实现多主机软件安装、服务启停、配置部署的自动化，运维效率提升 60%，配置错误率降至 0

        个人选课查询：展示已选课程列表，支持随时查看选课情况

  • 交互设计优化：设计清晰的控制台操作菜单，通过循环交互实现多轮操作，加入输入校验与错误提示，提升用户体验
  • 主机信息自动化采集：开发 configure_host.yml 脚本，利用 Jinja2 模板遍历所有主机，采集并输出各节点的 IP 地址与主机名，生成标准化配置文件 /tmp/hosts.txt
  • 系统测试与优化：完成功能测试用例设计，覆盖选课、退课、边界值等场景，修复课程编号输入异常、名额计算错误等问题，系统运行稳定
